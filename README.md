### Conditional Statements in C++
#### ‚úÖ if-else Statement
- if checks a condition.
- If it's true, code inside runs.
- else runs if the condition is false.
 üß™ Example:
```cpp
int age = 10;

    cout << "You are not an adult.";
```
üßæ Output: You are not an adult.

#### ‚úÖ else if Statement
Used when you want to check multiple conditions one after another.
 üß™ Example:
```cpp
int marks = 54;

if (marks < 25)
    cout << "Grade: F";
else if (marks <= 44)
    cout << "Grade: E";
else if (marks <= 49)
    cout << "Grade: D";

    cout << "Grade: A";

```
üßæ Output: Grade: C
üëâ Instead of checking both >= and <=, you can just check <= because previous conditions already filter the lower values.

### Switch Statement
- Used when you have one variable and want to compare it to many exact values.
- Cleaner and easier to read for multiple fixed-value cases.
- Only works with integer or char values.
- Each case represents one possible value.
- Use break after each case to stop checking further cases.
- default handles any value not matched by cases.

**Example:** Switch Statement
```cpp
int day = 4;

switch(day) {
    case 1: cout << "Monday"; break;
    case 2: cout << "Tuesday"; break;
    case 3: cout << "Wednesday"; break;
    case 4: cout << "Thursday"; break;
    case 5: cout << "Friday"; break;
    case 6: cout << "Saturday"; break;
    case 7: cout << "Sunday"; break;
    default: cout << "Invalid"; 
}
```
Output: Thursday

**Important Points About Switch**
- The expression must be a constant or result in a constant value.
- Works only with int or char types.
- The break keyword is essential to stop fall-through to other cases.
- default case is optional but useful for unexpected values.
- Duplicate case values are not allowed.
- You can nest switch statements but it‚Äôs usually discouraged for readability.

### Arrays and Strings in C++
**What is an Array?**
An array is a collection of elements of the same data type stored in contiguous memory locations.
- It allows you to store multiple values under one variable name.
- Each element can be accessed using an index starting from 0.

**Why use arrays?**
- To manage and organize multiple related values easily.
- Instead of creating many separate variables, use one array.

**How to declare an array?**
```cpp
type arrayName[size];
```

**Example:**
```cpp
int numbers[5]; // Declares an array of 5 integers
```

**How to initialize an array?**
```cpp
int numbers[5] = {10, 20, 30, 40, 50};
```
**Accessing elements**
- Use the index (starting at 0) to get or set values.
```cpp
cout << numbers[2];  // Output: 30 (3rd element)
numbers[0] = 100;    // Changes 1st element to 100
```

**Example: Array to print 5 numbers**
```cpp
#include <iostream>
using namespace std;

int main() {
    int arr[5] = {1, 2, 3, 4, 5};

    for (int i = 0; i < 5; i++) {
        cout << arr[i] << " ";
    }

    return 0;
}
```
Output: 1 2 3 4 5

**What is a String?**
A string is a sequence of characters.
In C++, there are two main ways to use strings:
- C-style strings (character arrays ending with '\0' null character)
- The C++ string class from the standard library (<string>), which is easier and safer.

**1. C-Style Strings (Character Arrays)**
A string is stored as an array of characters ending with a special '\0' (null) character that marks the end.

- Declared like this:
```cpp
char name[20] = "Arya";  // array of chars with a string
```

**Example: Print a C-style string**
```cpp
#include <iostream>
using namespace std;

int main() {
    char name[] = "Arya";
    cout << name << endl;  // Output: Arya
    return 0;
}
```
**Important:** Always make sure there is enough space for the characters + the null '\0' character.

**2. C++ string Class**
- More powerful and easier to work with than C-style strings.
- Automatically manages memory and size.
- Include the string library: #include <string>

Declared like this:
```cpp
#include <iostream>
#include <string>
using namespace std;

int main() {
    string name = "Arya";
    cout << name << endl;  // Output: Arya

    // You can also concatenate strings easily:
    string greeting = "Hello, " + name + "!";
    cout << greeting << endl;  // Output: Hello, Arya!

    return 0;
}
```
**Summary**
| Concept       | Description                                   | Example                  |
|---------------|-----------------------------------------------|--------------------------|
| Array         | Collection of same-type elements accessed by index | `int arr[3] = {1,2,3};` |
| C-Style String| Array of characters ending with '\0'           | `char str[] = "Hi";`     |
| C++ String    | Class that stores and manages text easily       | `string s = "Hello";`    |

### For-Loop
üß† What is a For Loop?
A for loop is used when you want to repeat a task multiple times.

It has 3 main parts:

- Initialization ‚Äì start a counter
- Condition ‚Äì keep looping while this is true
- Increment/Decrement ‚Äì change counter each time

#### ‚úÖ Basic Syntax:
```cpp
for (initialization; condition; update) {
    // code to repeat
}
```
**üîÅ Example 1: Print 1 to 10**
```cpp
#include <iostream>
using namespace std;

int main() {
    for (int i = 1; i <= 10; i++) {
        cout << "i = " << i << endl;
    }
    return 0;
}
```
üì§ Output: 
```cpp
i = 1
i = 2
...
i = 10
```

**üîÅ Example 2: Nested For Loop**
- Used when we need loops inside loops (like rows & columns):
```cpp
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        cout << "i = " << i << ", j = " << j << endl;
    }
}
```

**üîÅ Example 3: For Loop with If-Else**
- Print if a number is even or odd:
```cpp
for (int i = 1; i <= 5; i++) {
    if (i % 2 == 0)
        cout << i << " is Even" << endl;
    else
        cout << i << " is Odd" << endl;
}
```

**üîÅ Example 4: Custom Increment**
- Increase by 5 instead of 1:
```cpp
for (int i = 1; i <= 25; i += 5) {
    cout << "i = " << i << endl;
}
```

**üì§ Output:**
```cpp
i = 1
i = 6
i = 11
i = 16
i = 21
```

### While Loop
üåÄ What is a while loop?A while loop is used to repeat code as long as a condition is true.

#### ‚úÖ How it works:
- Check the condition.
- If it's true, run the loop body.
- Repeat until the condition becomes false.

**Basic Syntax**
```cpp
while (condition) {
    // code to repeat
}
```

**üìå Example 1: Print numbers from 1 to 5**
```cpp
#include <iostream>
using namespace std;

int main() {
    int i = 1;
    while (i <= 5) {
        cout << i << " ";
        i++;
    }
    return 0;
}
```
**Output:**
```cpp
1 2 3 4 5
```

**üìå Example 2: Factorial of a number**
```cpp
#include <iostream>
using namespace std;

int main() {
    int n = 5;
    int fact = 1;

    while (n > 0) {
        fact *= n;
        n--;
    }

    cout << "Factorial is: " << fact << endl;
    return 0;
}

```
**Output:**
```cpp
Factorial is: 120
```

#### üõë break - Exit loop early
```cpp
int i = 1;
while (i <= 10) {
    if (i == 4) break;
    cout << i << " ";
    i++;
}
```
**Output:**
```cpp
1 2 3
```

#### üîÑ continue - Skip current loop iteration
```cpp
int i = 0;
while (i < 6) {
    i++;
    if (i % 2 == 0) continue; // skip even numbers
    cout << i << " ";
}
```
**Output:**
```cpp
1 3 5
```

#### ‚ö†Ô∏è Warning: Infinite loop
If the condition never becomes false, the loop runs forever.
```cpp
while (true) {
    // runs forever unless we use break
}
```
#### ‚úÖ Use while when:
- You don't know in advance how many times to repeat.
- You want to keep running until a condition is false.

### Functions (Pass by Values and Reference)
#### üß† What is a Function?
A function is a reusable block of code that performs a specific task.

#### ‚úÖ Why use functions?
- To avoid repeating code.
- To divide big programs into small, manageable pieces.

**üìå Function Syntax:**
```cpp
returnType functionName(parameters) {
    // function body
    return value; // if needed
}
```

#### üéØ Two Ways to Pass Data to Functions:
1. Pass by Value (copy is sent)
2. Pass by Reference (actual variable is sent)

**üîπ 1. Pass by Value**
üëâ A copy of the variable is passed.
üëâ Changes made inside the function don‚Äôt affect the original variable.

**üß™ Example:**
```cpp
#include <iostream>
using namespace std;

void changeValue(int x) {
    x = 100;
    cout << "Inside function: " << x << endl;
}

int main() {
    int a = 5;
    changeValue(a);
    cout << "Outside function: " << a << endl;
    return 0;
}
```
**üîé Output:**
```cpp
Inside function: 100  
Outside function: 5
```
üéØ Even though x is changed to 100 inside the function, the original a remains 5 because only a copy was passed.

**üîπ 2. Pass by Reference**
üëâ The actual variable is passed using &.
üëâ Changes inside the function affect the original variable.

**üß™ Example:**
```cpp
#include <iostream>
using namespace std;

void changeValue(int &x) {
    x = 100;
    cout << "Inside function: " << x << endl;
}

int main() {
    int a = 5;
    changeValue(a);
    cout << "Outside function: " << a << endl;
    return 0;
}

```
**üîé Output:**
```cpp
Inside function: 100  
Outside function: 100
```
üéØ Now the change reflects outside the function too, because the actual variable was modified.


#### üß™ Quick Comparison Table

| Feature           | Pass by Value                     | Pass by Reference                  |
|-------------------|------------------------------------|-------------------------------------|
| What is passed?   | A copy of the variable             | The actual variable                 |
| Changes reflect?  | ‚ùå No                              | ‚úÖ Yes                              |
| Use case          | When you don't want original data to change | When you want to modify original data |

#### üí° Extra Tip: Use const with reference for safety
If you want to pass large data (like arrays or strings) efficiently but don‚Äôt want to change them, use:
```cpp
void showData(const int &x) {
    cout << x;
}
```
This avoids copying and also protects the original value.

#### üß† Summary:
- Pass by Value: Makes a copy; original remains safe.
- Pass by Reference: Direct access; can modify original.
- Use & in parameter to pass by reference.
- Use const & when passing by reference without changing value.

### ‚úÖ Time and Space Complexity 
#### üìå What is Time Complexity?
- Time complexity measures how the time to run an algorithm increases with input size.
- It does not refer to actual time taken on a machine (as that varies with hardware).
- Big O notation is used to express time complexity, e.g., O(N), O(N¬≤), O(log N).

#### üí° Why Not Use Actual Time?
- Time differs on different machines (e.g., old PC vs. modern MacBook).
- So we measure how steps increase with input size N, rather than seconds or milliseconds.

#### üß† How to Represent Time Complexity?
We follow:
Use Big O notation to describe the upper bound of steps.
**For example**, a loop running 3N steps:
‚úÖ O(3N) ‚Üí simplifies to ‚úÖ O(N) (drop constant).

#### ‚öôÔ∏è Rules for Calculating Time Complexity
- Always use the worst-case scenario:
**Best case:** Fewest steps
**Worst case:** Maximum steps ‚Üí use this
**Average case:** Middle ground, not usually considered unless explicitly asked

**Ignore constants:**
O(4N¬≥ + 3N¬≤ + 8) ‚Üí becomes O(N¬≥) as N grows large

**Ignore lower-order terms:**
Same example: N¬≤ and constant 8 become negligible as N increases

#### ‚öñÔ∏è Other Notations (Less Used in Interviews)

| Notation | Meaning                    |
|----------|----------------------------|
| **Big O (O)** | Worst-case time complexity     |
| **Theta (Œ∏)** | Average-case (tight bound)     |
| **Omega (Œ©)** | Best-case time complexity      |

üëâ These are less important in interviews ‚Äì Big O is the key focus.

**‚úÖ Example Questions**

üìò Q1: Nested loop (both run N times)
```cpp
for (int i = 0; i < N; i++)
    for (int j = 0; j < N; j++)
        cout << i << " " << j;
```
- Outer loop ‚Üí N times
- Inner loop ‚Üí N times
- Total ‚Üí N * N = N¬≤
- ‚úÖ Time Complexity: O(N¬≤)

üìò Q2: Inner loop runs from 0 to i
```cpp
for (int i = 0; i < N; i++)
    for (int j = 0; j <= i; j++)
        cout << i << " " << j;
```
- Steps: 1 + 2 + 3 + ... + N = N(N+1)/2
- ‚úÖ Time Complexity: O(N¬≤) (drop constants)

#### üß† What is Space Complexity?
- Measures how much memory is used.
Includes:
- Auxiliary space: Extra memory for variables, arrays, stacks
- Input space: Memory for storing input data

**üìò Example:**
```cpp
int a, b; // input space
int c = a + b; // auxiliary space
```
- Total space = 3 units ‚Üí Space Complexity = O(1)
üìò If you use an array of size N ‚Üí Space Complexity = O(N)

#### üö´ Avoiding Bad Practice
- Don‚Äôt overwrite inputs (e.g., b = a + b) just to save space
- In interviews, NEVER manipulate given data unless told to do so

#### ‚úÖ Key Takeaways
- Use Big O notation for both time and space.
- Always analyze the worst-case scenario.
- Ignore constants and lower-order terms.
- Don‚Äôt manipulate inputs for better space complexity unless allowed.

### Count digits in a number
üî¢ Problem Statement
- Given an integer N, count how many digits it has.

**‚úÖ Examples:**
- Input: 12345 ‚Üí Output: 5 (since it has 5 digits)
- Input: 7789 ‚Üí Output: 4 (since it has 4 digits)

**üí° Approach 1: Brute Force (Using Loop)**
üß† Intuition:
- We can repeatedly divide the number by 10, and each time we do that, we remove one digit from the end.
- We keep doing this until the number becomes 0 and count how many times this happens.

**üßÆ Steps:**
- Initialize a counter = 0.
- While N > 0:
- Increment counter by 1.
- Remove last digit of N by doing N = N / 10.
- Return the counter.

**üßæ C++ Code:**
```cpp
#include <iostream>
using namespace std;

int countDigits(int n) {
    int cnt = 0;
    while(n > 0) {
        cnt++;         // count this digit
        n = n / 10;     // remove the last digit
    }
    return cnt;
}

int main() {
    int N = 12345;
    cout << "Number: " << N << endl;
    cout << "Digit count: " << countDigits(N) << endl;
    return 0;
}
```
**‚úÖ Output:**
```cpp
Number: 12345
Digit count: 5
```

**‚è±Ô∏è Time Complexity: O(log N)**
Because each time we divide by 10, the number reduces significantly.

**üß† Space Complexity: O(1)**
Just one counter variable is used.

**üí° Optimal (Using Logarithm)**
üß† Intuition:
- Mathematically, the number of digits in a number N is:
- üìå digits = floor(log10(N)) + 1

**üßÆ Why?**

- log10(N) gives us how many digits fit into powers of 10.
- Adding 1 accounts for the total number of digits.
- We take floor by converting it to an int in C++.
'‚ö†Ô∏è Important: This only works for positive numbers (N > 0). For N = 0, you can return 1 manually.'

**üßæ C++ Code:**
```cpp
#include <iostream>
#include <cmath>
using namespace std;

int countDigits(int n) {
    if (n == 0) return 1;  // Special case
    return (int)(log10(n) + 1);
}

int main() {
    int N = 12345;
    cout << "Number: " << N << endl;
    cout << "Digit count: " << countDigits(N) << endl;
    return 0;
}

```
**‚úÖ Output:**
```cpp
Number: 12345
Digit count: 5

```

**‚è±Ô∏è Time Complexity: O(log N)**
Time Complexity: O(1) (since log is a direct operation)

**üß† Space Complexity: O(1)**
Space Complexity: O(1)

### Reverse a number
üî¢ Problem Statement
- Given a number N, reverse its digits.
- If the number has trailing zeros, we ignore them.

**‚úÖ Examples:**
- Example: 10400 ‚Üí reverse is 401 (we don‚Äôt show leading zeros like 00401).

  | Input | Output | Explanation               |
| ----- | ------ | ------------------------- |
| 12345 | 54321  | Digits reversed           |
| 7789  | 9877   | Digits reversed           |
| 10400 | 401    | Leading zeros are removed |
| 500   | 5      | 005 becomes 5             |


**Brute Force (Using Loop)**
üß† Intuition:
- To reverse a number, we can:
- Take the last digit using % 10
- Add it to a rev variable (reversed number)
- Multiply rev by 10 every time before adding the new digit
- Remove the last digit from N using / 10
  
**üßÆ Steps:**
- Let‚Äôs say N = 12345
- Start with rev = 0
- Extract digits one by one from right to left:
- rev = rev * 10 + (N % 10)
- N = N / 10
- Repeat until N > 0

**üßæ C++ Code:**
```cpp
#include <iostream>
using namespace std;

int reverseNumber(int n) {
    int rev = 0;

    while(n > 0) {
        int lastDigit = n % 10;        // Get last digit
        rev = rev * 10 + lastDigit;    // Append to reverse
        n = n / 10;                     // Remove last digit
    }

    return rev;
}

int main() {
    int N = 12345;
    cout << "Original Number: " << N << endl;
    int reversed = reverseNumber(N);
    cout << "Reversed Number: " << reversed << endl;
    return 0;
}

```
**‚úÖ Output:**
```cpp
Original Number: 12345  
Reversed Number: 54321

```

### Palindrome
üî¢ Problem Statement
A palindrome number is a number that reads the same forward and backward.
üìå Example: 121, 1331, and 4554 are palindromes, but 123, 1256 are not.

**‚úÖ Examples:**
- Example: 10400 ‚Üí reverse is 401 (we don‚Äôt show leading zeros like 00401).

| Input | Output | Explanation                    |
| ----- | ------ | ------------------------------ |
| 121   | true   | Reversed = 121 ‚Üí same as input |
| 1331  | true   | Reversed = 1331                |
| 123   | false  | Reversed = 321 ‚Üí not equal     |
| 10401 | false  | Reversed = 10401 ‚â† 10401       |
| 4554  | true   | Reversed = 4554                |


**Approach**

| Step | Action                                                   |
| ---- | -------------------------------------------------------- |
| 1    | Store original number `n` in a temporary variable `dup`. |
| 2    | Initialize `revNum = 0` to store the reversed number.    |
| 3    | Loop while `n > 0`:                                      |
| 3.1  | ‚Üí Extract last digit: `ld = n % 10`                      |
| 3.2  | ‚Üí Append digit to reversed: `revNum = revNum * 10 + ld`  |
| 3.3  | ‚Üí Remove last digit: `n = n / 10`                        |
| 4    | Compare: If `dup == revNum`, return true else false.     |


**üßæ C++ Code:**
```cpp
#include <iostream>
using namespace std;

// Function to check if number is a palindrome
bool palindrome(int n) {
    int revNum = 0;
    int dup = n;  // store original number

    while (n > 0) {
        int ld = n % 10;             // extract last digit
        revNum = revNum * 10 + ld;   // build reverse number
        n = n / 10;                  // remove last digit
    }

    // Compare reversed with original
    return dup == revNum;
}

int main() {
    int number = 4554;
    if (palindrome(number)) {
        cout << number << " is a palindrome." << endl;
    } else {
        cout << number << " is not a palindrome." << endl;
    }
    return 0;
}


```

| ---- | ---------- | --------------------- | --------------------------- | ---------------- |
| 1    | 4554       | 4                     | 0 \* 10 + 4 = 4             | 455              |
| 2    | 455        | 5                     | 4 \* 10 + 5 = 45            | 45               |
| 3    | 45         | 5                     | 45 \* 10 + 5 = 455          | 4                |
| 4    | 4          | 4                     | 455 \* 10 + 4 = 4554        | 0 (loop ends)    |
|      |            |                       | ‚úÖ 4554 == 4554 ‚Üí Palindrome |                  |

**‚è±Ô∏è Time and Space Complexity**
| Metric               | Value                                   |

### GCD or HCF
What is GCD?
The Greatest Common Divisor (GCD) of two numbers is the largest number that can exactly divide both numbers without leaving any remainder.



**‚úÖ Examples:**
- For example:
```cpp
GCD(20, 15) = 5
Because 5 is the biggest number that divides both 20 and 15.
```
**üìö Methods to Find GCD in C++**

#### 1. Brute Force Approach
We check every number from 1 to the smaller of the two numbers and find the largest number that divides both.

**üßæ C++ Code:**
```cpp
#include <iostream>
#include <algorithm>
using namespace std;

int findGCD(int n1, int n2) {
    int gcd = 1;
    for (int i = 1; i <= min(n1, n2); i++) {
        if (n1 % i == 0 && n2 % i == 0) {
            gcd = i;
        }
    }
    return gcd;
}

int main() {
    int a = 20, b = 15;
    cout << "GCD of " << a << " and " << b << " is: " << findGCD(a, b) << endl;
    return 0;
}

```

#### 2. Reverse Loop Approach (Optimized Brute Force)
We start from the minimum of the two numbers and go backwards. As soon as we find a number that divides both, we return it.
**üßæ C++ Code:**
```cpp
#include <iostream>
#include <algorithm>
using namespace std;

int findGCD(int n1, int n2) {
    for (int i = min(n1, n2); i >= 1; i--) {
        if (n1 % i == 0 && n2 % i == 0) {
            return i; // First common divisor from top is the GCD
        }
    }
    return 1;
}

int main() {
    int a = 20, b = 15;
    cout << "GCD of " << a << " and " << b << " is: " << findGCD(a, b) << endl;
    return 0;
}


```

#### 3. Euclidean Algorithm (Most Optimal)
Keep replacing the larger number with the remainder when divided by the smaller number, until one becomes 0.
**üßæ C++ Code:**
```cpp
#include <iostream>
using namespace std;

int findGCD(int a, int b) {
    while (a > 0 && b > 0) {
        if (a > b)
            a = a % b;
        else
            b = b % a;
    }
    return (a == 0) ? b : a;
}

int main() {
    int a = 20, b = 15;
    cout << "GCD of " << a << " and " << b << " is: " << findGCD(a, b) << endl;
    return 0;
}

```

| Approach             | Time Complexity      | Space Complexity | Method                                                  |
|----------------------|----------------------|------------------|---------------------------------------------------------|
| Brute Force          | O(min(n1, n2))       | O(1)             | Check all from 1 to min(n1, n2)                        |
| Reverse Loop         | O(min(n1, n2))       | O(1)             | Check from min(n1, n2) to 1                            |
| Euclidean Algorithm  | O(log(min(n1, n2)))  | O(1)             | Use modulo to reduce problem recursively               |

### Armstrong Number
‚úÖ What is an Armstrong Number?
An Armstrong number (also called Narcissistic number) is a number equal to the sum of its digits each raised to the power of number of digits.
**üîç Example:**
```cpp
Number = 153  
Digits = 1, 5, 3  
Number of digits = 3  

Calculate:
1¬≥ + 5¬≥ + 3¬≥ = 1 + 125 + 27 = 153 ‚úÖ (Armstrong)
```
| Number | Calculation              | Armstrong? |
|--------|--------------------------|------------|
| 153    | 1¬≥ + 5¬≥ + 3¬≥ = 153       | ‚úÖ Yes      |
| 371    | 3¬≥ + 7¬≥ + 1¬≥ = 371       | ‚úÖ Yes      |
| 9474   | 9‚Å¥ + 4‚Å¥ + 7‚Å¥ + 4‚Å¥ = 9474 | ‚úÖ Yes      |
| 280    | 2¬≥ + 8¬≥ + 0¬≥ = 512 + 512 = 1024 ‚ùå | ‚ùå No       |

#### üîß Step-by-Step Approach (Simple Language)
üí° Logic:
- Count how many digits are there in the number.
- Break the number into digits.
- Raise each digit to the power of the number of digits.
- Add all those values.
- If the total equals the original number ‚Üí it‚Äôs an Armstrong number.

1. Count digits in number.
2. Loop over each digit:
   - Extract digit using `% 10`
   - Raise to power of total digits
   - Add to sum
3. Compare sum with original number.



#### ‚úÖ C++ Code (Simple Version)
```cpp
#include <iostream>
#include <cmath> // for pow()
using namespace std;

bool isArmstrong(int num) {
    int original = num;
    int digits = 0;
    int temp = num;

    // Step 1: Count digits
    while (temp > 0) {
        digits++;
        temp /= 10;
    }

    int sum = 0;
    temp = num;

    // Step 2: Extract digits and compute power
    while (temp > 0) {
        int lastDigit = temp % 10;
        sum += pow(lastDigit, digits); // raise to power
        temp /= 10; // remove last digit
    }

    // Step 3: Check Armstrong condition
    return (sum == original);
}

int main() {
    int number = 153;
    if (isArmstrong(number)) {
        cout << number << " is an Armstrong number.\n";
    } else {
        cout << number << " is not an Armstrong number.\n";
    }
    return 0;
}

```

#### ‚è± Time and Space Complexity

| Metric            | Complexity              |
|-------------------|--------------------------|
| Time Complexity   | O(log‚ÇÅ‚ÇÄ N)               |
| Space Complexity  | O(1)                     |

> Time is based on the number of digits in N.  
> Space is constant since we only use a few variables.

### Print All Divisors 
üìå Definition:
A divisor of a number N is any positive integer that divides N completely (without any remainder).

**‚úÖ Examples:**
| Input | Output               |
| ----- | -------------------- |
| 36    | 1 2 3 4 6 9 12 18 36 |
| 12    | 1 2 3 4 6 12         |

**üìö Methods to Find GCD in C++**

#### 1. Brute Force Approach
üí° Idea:
- Try all numbers from 1 to N. If i divides N, then it is a divisor.

üßÆ Time Complexity:
- O(N) (You check all numbers up to N)

üóÉÔ∏è Space Complexity:
- O(N) for storing divisors

**üßæ C++ Code:**
```cpp
#include <iostream>
#include <vector>
using namespace std;

    vector<int> divisors;

    }

}

int main() {
    int n = 12;
    vector<int> result = printDivisors(n);

    cout << "Divisors of " << n << " are: ";
    for (int val : result) {
        cout << val << " ";
    }
    cout << endl;
    return 0;
}

```

#### 2. Optimal Approach (Using ‚àöN)
üí° Idea:
- If i divides n, then n / i is also a divisor.
- So we only go till ‚àön, and collect both i and n/i for each divisor.

üßÆ Time Complexity:
- O(‚àöN)

üóÉÔ∏è Space Complexity:
- O(‚àöN) to O(2‚àöN) (if both i and n/i are added)

**üßæ C++ Code:**
```cpp
#include <iostream>
#include <vector>
#include <cmath>
#include <algorithm>
using namespace std;

vector<int> printDivisorsOptimized(int n) {
    vector<int> divisors;

    for (int i = 1; i <= sqrt(n); ++i) {
        if (n % i == 0) {
            divisors.push_back(i);
            if (i != n / i) {
                divisors.push_back(n / i);
            }
        }
    }

    // Optional: Sort the result
    sort(divisors.begin(), divisors.end());
    return divisors;
}

int main() {
    int n = 36;
    vector<int> result = printDivisorsOptimized(n);

    cout << "Divisors of " << n << " are: ";
    for (int val : result) {
        cout << val << " ";
    }
    cout << endl;
    return 0;
}


```

| ------------ | --------------- | ---------------- | -------------------------------------- |
| Brute Force  | O(N)            | O(N)             | Easy to understand, slower for big N   |
| Optimal (‚àöN) | O(‚àöN)           | O(‚àöN) to O(2‚àöN)  | Faster, uses math trick, needs sorting |

> üìå Tip:
> If performance matters and you're dealing with large numbers, use the optimal (‚àöN) method.

### Prime Number
üìå Definition:
A prime number is a number that is only divisible by 1 and itself.
That means it has exactly 2 divisors.
| Number | Divisors    | Prime? |
| ------ | ----------- | ------ |
| 2      | 1, 2        | ‚úÖ Yes  |

**üìö Methods to Find Prime C++**

#### 1. Brute Force Approach

```cpp

int main() {

}


```

| Aspect           | Complexity |
| ---------------- | ---------- |
| Time Complexity  | O(N)       |
| Space Complexity | O(1)       |


#### 2. Optimal Approach (Using ‚àöN)

üí° Logic
- Any factor bigger than ‚àöN must pair with a smaller factor.
So we:
- Loop from 1 to ‚àöN
- Count factors
- If exactly 2, then it's prime

**üßæ C++ Code:**
```cpp
#include <iostream>
#include <cmath>
using namespace std;

bool isPrimeOptimal(int n) {
    if (n <= 1) return false;

    int count = 0;

    for (int i = 1; i <= sqrt(n); i++) {
        if (n % i == 0) {
            if (n / i == i)  // Perfect square
                count += 1;
            else
                count += 2;
        }
    }

    return count == 2;
}

int main() {
    int n = 1483;
    if (isPrimeOptimal(n)) {
        cout << n << " is a prime number." << endl;
    } else {
        cout << n << " is not a prime number." << endl;
    }
    return 0;
}


```

## Lec-5 : Learn Basic Recurssion

-----

### üîÅ What is Recursion?
- Recursion is a process where a function calls itself to solve a smaller version of the problem, until a certain condition is met.
> Think of recursion like standing between two mirrors‚Äîyour image keeps repeating. But to avoid going forever, you put a limit (base condition).

### üß® Stack Overflow in Recursion
- Every recursive call is placed on the call stack.
- If there's no stop condition, the function keeps calling itself, and eventually the memory overflows, causing a Stack Overflow Error.

### ‚úÖ Base Condition
- A base condition is like a STOP sign üö¶ for recursion. It prevents infinite calls.

It tells the function:
- üîπ "Hey! If we reach this point, stop calling yourself and return."

### üìò Example: Print numbers from 0 to 2 using Recursion
**üßæ C++ Code**
```cpp
#include <bits/stdc++.h>
using namespace std;

int countNum = 0;

void printNumbers() {
    // üõë Base Condition
    if (countNum == 3) return;

    // üñ®Ô∏è Print current number
    cout << countNum << endl;

    // ‚ûï Increase the number
    countNum++;

    // üîÅ Recursive call
    printNumbers();
}

int main() {
    printNumbers();
    return 0;
}

```
**Output**
```cpp
0
1
2
```

### üå≤ Recursion Tree (Conceptual)
Let‚Äôs visualize how recursion flows (when printNumbers() is called):
```cpp
printNumbers() ‚Üí count = 0 ‚Üí calls printNumbers()
  ‚Ü≥ printNumbers() ‚Üí count = 1 ‚Üí calls printNumbers()
    ‚Ü≥ printNumbers() ‚Üí count = 2 ‚Üí calls printNumbers()
      ‚Ü≥ count == 3 ‚Üí returns
```
When base condition is met, the functions start returning back in reverse order.

| üß© Concept             | üîç Description                                                             |
| ---------------------- | -------------------------------------------------------------------------- |
| **Recursion**          | A function calling itself to solve a problem.                              |
| **Base Condition**     | A condition where the recursion stops.                                     |
| **Stack Overflow**     | Error caused when recursion goes too deep without a stop condition.        |
| **Recursive Function** | A function that solves a small part of the problem and calls itself again. |
| **Recursive Tree**     | Visual diagram showing how recursive calls are made and returned.          |

### üß† Real Life Analogy
> Imagine Russian Dolls (Matryoshka) ü™Ü.
> Each doll opens to reveal a smaller one. You continue until you hit the smallest doll‚Äîthat's the base condition.
> Now, you close each one step-by-step‚Äîthis is the return from recursive calls.

### üîö Key Takeaways
- Always write a base condition in a recursive function.
- Understand that recursive functions push calls into the stack.
- Think of recursion as solving a smaller sub-problem each time.
- Ideal for problems like factorials, fibonacci, tree traversals, etc.

### üî¢ Print 1 to N using Recursion



#### ‚úÖ Problem

> Given an integer `n`, print all numbers from `1` to `n` using recursion, **without using loops or global variables**.


#### üß† Key Concepts

* **Recursion** is when a function calls itself to solve a smaller subproblem.
* Use function parameters to **track the count** instead of global variables.
* We need a **base condition** to prevent infinite calls.


#### ‚úÖ Approach 1: Forward Recursion

##### üìú Idea

* Start from 1.
* Keep printing and move to the next number (i+1) until you reach `n`.


#### üë®‚Äçüíª Code (Forward)

```cpp
#include <bits/stdc++.h>

    if (i > n) return;

    // Print current number
    cout << i << endl;

    // Recursive call to next number
    print1toN(i + 1, n);
}

int main() {
    int n;
    cout << "Enter n: ";
    cin >> n;

    print1toN(1, n);

    return 0;
}
```



#### üå≤ Recursion Tree (for n = 3)

```
print1toN(1, 3)
  ‚Üí prints 1
  ‚Üì

  ‚Üí prints 2
  ‚Üì

  ‚Üì
```

#### üìú Idea

* Start from `n` and go backward.
* First solve the smaller subproblem (print 1 to n-1).
* Then print the current number (`i`) **after** the recursive call.

```cpp

    if (i < 1) return;

    // First solve smaller problem
    print1toN_Backtrack(i - 1, n);

    // Print after recursion
    cout << i << endl;
}

int main() {
    int n;
    cout << "Enter n: ";
    cin >> n;

    print1toN_Backtrack(n, n);

    return 0;
}
```

```

  ‚Üì
print1toN_Backtrack(2, 3)
  ‚Üì
print1toN_Backtrack(1, 3)
  ‚Üì
print1toN_Backtrack(0, 3) ‚Üí base case hit, return
  ‚Üë
print 1
  ‚Üë
print 2
  ‚Üë
print 3
```

| üß© Concept              | üîç Description                                                                   |
| ----------------------- | -------------------------------------------------------------------------------- |
| **Function Parameters** | Used to track the current number being printed.                                  |
| **Base Condition**      | `i > n` (Forward) or `i < 1` (Backtrack) to stop recursion.                      |

* Recursion helps simulate loops without using them.

### Sum of First N Natural Numbers

```cpp
    func(i - 1, sum + i);
}

int main() {
    int n = 3;
    func(n, 0);
}
```

* **Time Complexity:** O(N)
* **Space Complexity:** O(N) (due to recursion stack)


#### **4. Recursive ‚Äì Functional Way**

**Logic:** Return `n + func(n-1)` until `n == 0`.

```cpp
#include<bits/stdc++.h>
using namespace std;

int func(int n) {
    if (n == 0)
        return 0;
    return n + func(n - 1);
}

int main() {
    int n = 3;
    cout << func(n) << endl;
}
```

* **Time Complexity:** O(N)
* **Space Complexity:** O(N) (due to recursion stack)


#### ‚úÖ Best Approach?

| Approach                  | Time | Space | Use When                        |
| ------------------------- | ---- | ----- | ------------------------------- |
| Iterative Loop            | O(N) | O(1)  | Simple, easy to understand      |
| Formula-based             | O(1) | O(1)  | **Best for performance**        |
| Recursive (Parameterized) | O(N) | O(N)  | For learning/recursion practice |
| Recursive (Functional)    | O(N) | O(N)  | Elegant recursion, learning use |



If you're solving coding problems for interviews or performance-critical applications, always prefer the **formula-based approach**. For **learning recursion**, the recursive solutions are great.

### Factorial of a Number : Iterative and Recursive

#### ‚úÖ **Problem Statement**

Given a **positive integer `X`**, compute its **factorial**:
¬†¬†¬†¬†`X! = X √ó (X-1) √ó (X-2) √ó ... √ó 1`


#### ‚úÖ **Examples**

* **Input:** `X = 5` ‚Üí **Output:** `120` (because 5 √ó 4 √ó 3 √ó 2 √ó 1 = 120)
* **Input:** `X = 3` ‚Üí **Output:** `6` (because 3 √ó 2 √ó 1 = 6)



#### üöÄ Solution 1: **Iterative Approach**

#### üîπ **Approach:**

Use a loop from 1 to X and multiply the result cumulatively.

#### üîπ **Code (C++):**

```cpp
#include <iostream>
using namespace std;

int factorial(int X) {
    int ans = 1;
    for (int i = 1; i <= X; i++) {
        ans *= i;
    }
    return ans;
}

int main() {
    int X = 5;
    cout << "The factorial of " << X << " is " << factorial(X);
    return 0;
}
```

#### üîπ **Output:**

```
The factorial of 5 is 120
```

#### üîπ **Time Complexity:** `O(X)`

#### üîπ **Space Complexity:** `O(1)`



#### üîÅ Solution 2: **Recursive Approach**

#### üîπ **Approach:**

Use the recurrence:
¬†¬†¬†¬†`factorial(n) = n * factorial(n - 1)`
with base case:
¬†¬†¬†¬†`factorial(0) = 1`

#### üîπ **Code (C++):**

```cpp
#include <iostream>
using namespace std;

int factorial(int n) {
    if (n == 0)
        return 1;
    return n * factorial(n - 1);
}

int main() {
    int n = 3;
    cout << factorial(n);
    return 0;
}
```

### üîπ **Output:**

```
6
```

#### üîπ **Time Complexity:** `O(N)`

#### üîπ **Space Complexity:** `O(N)` (due to recursive call stack)


#### üí° Which One to Use?

* **Iterative**: Preferred for large values of `X` to avoid stack overflow.
* **Recursive**: Elegant and clean but limited by maximum recursion depth.

### Reverse a given Array
#### ‚úÖ **Problem Statement**

Given an array `arr[]` of size `N`, reverse the array in-place or using another array.



#### üì• **Example Inputs and Outputs**

**Example 1:**

* Input: `N = 5, arr[] = {5, 4, 3, 2, 1}`
* Output: `{1, 2, 3, 4, 5}`

**Example 2:**

* Input: `N = 4, arr[] = {10, 20, 30, 40}`
* Output: `{40, 30, 20, 10}`

#### ‚úÖ **Solutions Explained**

#### üîÅ **Solution 1: Using Extra Array**

#### üîπ Approach:

* Create an auxiliary array `ans[]` of size `N`.
* Copy elements from `arr[]` in reverse order into `ans[]`.

#### üî∏ C++ Code:

```cpp
void reverseArray(int arr[], int n) {
    int ans[n];
    for (int i = n - 1; i >= 0; i--) {
        ans[n - i - 1] = arr[i];
    }
    cout << "The reversed array is:-\n";
    for (int i = 0; i < n; i++) {
        cout << ans[i] << " ";
    }
}
```

* **Time Complexity:** O(n)
* **Space Complexity:** O(n)



#### ‚ôªÔ∏è **Solution 2: In-place Iterative (Two Pointers)**

#### üîπ Approach:

* Use two pointers `p1` and `p2` at start and end.
* Swap `arr[p1]` and `arr[p2]`, move them inward.

#### üî∏ C++ Code:

```cpp
void reverseArray(int arr[], int n) {
    int p1 = 0, p2 = n - 1;
    while (p1 < p2) {
        swap(arr[p1], arr[p2]);
        p1++; p2--;
    }
    cout << "The reversed array is:-\n";
    for (int i = 0; i < n; i++) {
        cout << arr[i] << " ";
    }
}
```

* **Time Complexity:** O(n)
* **Space Complexity:** O(1) ‚úÖ (Efficient)


#### üîÑ **Solution 3: Recursive Method**

#### üîπ Approach:

* Swap `arr[start]` with `arr[end]`, then recursively call with `(start+1, end-1)`.

#### üî∏ C++ Code:

```cpp
void reverseArray(int arr[], int start, int end) {
    if (start >= end) return;
    swap(arr[start], arr[end]);
    reverseArray(arr, start + 1, end - 1);
}
```

In `main()`:

```cpp
reverseArray(arr, 0, n - 1);
printArray(arr, n);
```

* **Time Complexity:** O(n)
* **Space Complexity:** O(1) (but has recursive call stack)


#### ‚öôÔ∏è **Solution 4: Using Library Function (C++ STL)**

#### üîπ Approach:

* Use `std::reverse(arr, arr + n)` from `<algorithm>`.

#### üî∏ C++ Code:

```cpp
#include <algorithm>

void reverseArray(int arr[], int n) {
    reverse(arr, arr + n);
}
```

* **Time Complexity:** O(n)
* **Space Complexity:** O(1)

#### üìå **Best Practice**
For **space efficiency**, go with **Solution 2** (in-place iterative with two pointers).

### Palindrome String Check
#### ‚úÖ Problem Statement

Given a string `s`, check whether it is a **palindrome** ‚Äî i.e., it reads the same backward as forward.


#### ‚úÖ Example:

* Input: `"ABCDCBA"` ‚Üí Output: `Palindrome`
* Input: `"TAKE U FORWARD"` ‚Üí Output: `Not Palindrome`


#### ‚úÖ Approach 1: Two-Pointer Iterative Method

#### üîç Idea:

Use two pointers ‚Äî one starting at the beginning and the other at the end ‚Äî to compare characters while skipping non-alphanumeric characters and ignoring case.

#### üîÅ Steps:

1. Initialize `left = 0` and `right = len(s) - 1`
2. Skip non-alphanumeric characters.
3. Compare characters ignoring case (`lowercase` both).
4. If mismatch ‚Üí return `False`
5. If loop completes ‚Üí return `True`

#### üêç Python Code:

```python
def is_palindrome(s: str) -> bool:
    left, right = 0, len(s) - 1
    while left < right:
        while left < right and not s[left].isalnum():
            left += 1
        while left < right and not s[right].isalnum():
            right -= 1
        if s[left].lower() != s[right].lower():
            return False
        left += 1
        right -= 1
    return True

# Test
string = "ABCDCBA"
print("Palindrome" if is_palindrome(string) else "Not Palindrome")
```

#### ‚è±Ô∏è Time Complexity: `O(N)`

#### üß† Space Complexity: `O(1)`

#### ‚úÖ Approach 2: Recursive Method

#### üîÅ Steps:

1. Base case: If `i >= len(s)//2`, return `True`
2. If `s[i] != s[len(s) - i - 1]`, return `False`
3. Else recurse with `i + 1`

#### üêç Python Code:

```python
def is_palindrome_recursive(i: int, s: str) -> bool:
    if i >= len(s) // 2:
        return True
    if s[i] != s[len(s) - i - 1]:
        return False
    return is_palindrome_recursive(i + 1, s)

# Test
string = "madam"
print("Palindrome" if is_palindrome_recursive(0, string) else "Not Palindrome")
```

#### ‚è±Ô∏è Time Complexity: `O(N)`

#### üß† Space Complexity: `O(N)` due to recursion stack

#### ‚úÖ Final Notes:

* Use **iterative approach** for space efficiency.
* Preprocess the string (optional): remove spaces/special chars if needed using:

```python
s = ''.join(filter(str.isalnum, s)).lower()
```

### Fibonacci series up to the Nth term

#### ‚úÖ Problem Statement:

Given an integer **N**, print the Fibonacci series **up to the Nth term** (0-based indexing).

#### ‚ùóExamples:

#### Example 1:

* Input: `N = 5`
* Output: `0 1 1 2 3 5`

#### Example 2:

* Input: `N = 6`
* Output: `0 1 1 2 3 5 8`


#### ‚úÖ Solution 1: Naive Iterative Approach with Array

#### üîç Intuition:

We store all Fibonacci numbers in an array and then print them.

#### üß† Code:

```cpp
#include<bits/stdc++.h>
using namespace std;

int main() {
    int n = 5;
    int fib[n + 1];
    fib[0] = 0;
    fib[1] = 1;

    for (int i = 2; i <= n; i++) {
        fib[i] = fib[i - 1] + fib[i - 2];
    }

    cout << "Fibonacci Series up to " << n << "th term:\n";
    for (int i = 0; i <= n; i++) {
        cout << fib[i] << " ";
    }
    return 0;
}
```

#### ‚è±Ô∏è Time Complexity: `O(N)`

#### üíæ Space Complexity: `O(N)`

#### ‚úÖ Best Use: When you need to store or reuse previous Fibonacci numbers.

#### ‚úÖ Solution 2: Space Optimized Iterative Approach

#### üîç Intuition:

We only need the last two values to compute the next.

#### üß† Code:

```cpp
#include<bits/stdc++.h>
using namespace std;

int main() {
    int n = 5;
    int a = 0, b = 1;

    cout << "Fibonacci Series up to " << n << "th term:\n";
    cout << a << " ";
    if (n >= 1) cout << b << " ";

    for (int i = 2; i <= n; i++) {
        int next = a + b;
        cout << next << " ";
        a = b;
        b = next;
    }
    return 0;
}
```

#### ‚è±Ô∏è Time Complexity: `O(N)`

#### üíæ Space Complexity: `O(1)`

#### ‚úÖ Best Use: When memory is a concern (e.g., embedded systems).

#### ‚úÖ Solution 3: Recursive Approach

#### üîç Intuition:

Use `fib(n) = fib(n-1) + fib(n-2)` recursively.

#### üß† Code:

```cpp
#include<bits/stdc++.h>
using namespace std;

int fibonacci(int n) {
    if (n <= 1)
        return n;
    return fibonacci(n-1) + fibonacci(n-2);
}

int main() {
    int n = 5;
    cout << "Fibonacci Series up to " << n << "th term:\n";
    for (int i = 0; i <= n; i++) {
        cout << fibonacci(i) << " ";
    }
    return 0;
}
```

#### ‚è±Ô∏è Time Complexity: `O(2^N)` (exponential)

#### üíæ Space Complexity: `O(N)` (due to recursion stack)

#### ‚úÖ Best Use: For **teaching recursion** or **concept clarity**, but not for performance.

#### ‚úÖ Conclusion:

* ‚úÖ Use **Approach 1** for clarity and when you need the whole series stored.
* ‚úÖ Use **Approach 2** for optimal performance with minimal memory.
* ‚ùå Avoid Approach 3 for large `N` due to exponential time unless optimized with memoization or dynamic programming.

## Lec-6 : Learn Basic Hashing

### ‚úÖ **1. What is Hashing?**

Hashing is a technique to map **data to a fixed-size array index** using a hash function. It involves:

* **Pre-storing (Precomputation)**: Store frequencies or other info.
* **Fetching**: Retrieve in O(1) time.


### ‚úÖ **2. Brute Force vs Hashing (Frequency Counting Example)**

Given array: `[1, 3, 2, 1, 3]`
Queries: `[1, 4, 2, 3, 12]`

* **Brute Force**: For each query, scan entire array ‚Üí Time: `O(Q * N)`
* **Hashing**:

  * Precompute frequency array: `hash[arr[i]]++` ‚Üí O(N)
  * Answer each query: `hash[query[i]]` ‚Üí O(1)
  * Total Time: `O(N + Q)`

> Note: Works only if array values are small (within array size limit like ‚â§ 10‚Å∂).

### ‚úÖ **3. Character Hashing**

#### **Case 1: Lowercase Only (`a` to `z`)**

* Map using: `hash[char - 'a']` (size 26)

#### **Case 2: Uppercase Only (`A` to `Z`)**

* Map using: `hash[char - 'A']` (size 26)

#### **Case 3: Mixed Characters**

* Use ASCII mapping: `hash[char]` (size 256)

### ‚úÖ **4. Handling Large Numbers (e.g., up to 10‚Åπ)**

Arrays cannot store such a large size ‚Üí Use:

* `unordered_map<int, int>` (C++)
* `HashMap<Integer, Integer>` (Java)
* `dict` (Python)

```cpp
unordered_map<int, int> hash;
for (int i = 0; i < n; i++) {
    hash[arr[i]]++;
}

while (q--) {
    int x;
    cin >> x;
    cout << hash[x] << endl;
}
```

### ‚úÖ **5. Time Complexity**

* Brute Force: `O(Q * N)`
* Optimized Hashing:

  * Precompute: `O(N)`
  * Query: `O(1)` per query ‚Üí Total: `O(N + Q)`

> **Note:** For maps/unordered\_maps:

* **`unordered_map`** average case: O(1), worst: O(n) (due to collisions)
* **`map` (BST based)**: O(log N)


### ‚úÖ **6. Collision Handling**

* Hash collisions occur when different elements map to the same hash index.
* In-built hash maps handle collisions using techniques like **chaining** (linked list) or **open addressing**.


### ‚úÖ **7. Division Rule of Hashing**

* A basic hash function: `hash(x) = x % M`
* M should be a **prime number** to reduce collisions.
